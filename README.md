# Smoothie

Mobile-first AI agent controller. Your Mac runs the agents, your iPhone controls them.

```
┌──────────────┐  Tailscale    ┌──────────────┐  PTY / HTTP    ┌─────────────┐
│  iPhone app  │ ◄──────────►  │  Mac daemon  │ ◄────────────► │  CLI agent  │
│  (SwiftUI)   │   HTTP + SSE  │  (Swift)     │                │  (opencode  │
│              │               │              │                │   claude)   │
└──────────────┘               └──────────────┘                └─────────────┘
```

All Swift, both sides. No cloud, no JavaScript, no third-party services. Tailscale is the only networking layer.

## Requirements

- macOS with Tailscale installed (`brew install --cask tailscale`)
- iPhone with the Tailscale app installed, signed into the same tailnet
- At least one CLI agent on PATH: `opencode`, `claude` (Anthropic Claude Code)
- Xcode 16+ (for the iOS app)
- Swift 6.0+ (for the server)

## Three-step start

### 1 — Run the server on your Mac

```bash
cd packages/server
swift run
```

The server prints its bind address. Note it. If Tailscale isn't running it falls back to `127.0.0.1:7749` and warns.

### 2 — Build the iOS app

```bash
cd packages/mobile
./setup.sh              # installs xcodegen via Homebrew if needed
open Smoothie.xcodeproj
```

In Xcode, select your iPhone (or simulator) and Cmd-R.

### 3 — Connect

Open Smoothie on your iPhone. Enter the server address from step 1 (e.g. `100.64.0.10` or `100.64.0.10:7749`). You're in.

## What's in MVP

- **Adapters:** OpenCode (HTTP via `opencode serve`), Claude Code (PTY via `claude -p --output-format stream-json`).
- **iOS app:** Connect → home (projects + active sessions) → new session (pick project + CLI) → live session (event stream, send messages, kill). Local notifications when the agent flips to `waiting` or `done`.
- **Server:** Sleep prevention via IOKit (system stays awake while sessions run), Tailscale-only binding, project-path validation, single-binary install.

Gemini and Codex adapters are wired into the protocol but throw `not implemented` — the registry still reports them so the iOS picker shows them greyed out.

## Architecture

Two packages:

- [`packages/server`](./packages/server) — Swift Package, one external dep (Hummingbird 2.x).
- [`packages/mobile`](./packages/mobile) — iOS app, SwiftUI, iOS 17+, Xcode project generated by xcodegen.

See [`AGENTS.md`](./AGENTS.md) for layout and contribution rules.

## Development commands

```bash
# Server
cd packages/server
swift build                # debug
swift build -c release     # release
swift run                  # run

# iOS app
cd packages/mobile
./setup.sh                 # one-time setup
xcodegen generate          # re-run after editing project.yml or adding files
xcodebuild -project Smoothie.xcodeproj -scheme Smoothie \
  -destination 'platform=iOS Simulator,name=iPhone 17' build
```

## Install the server as a LaunchAgent (optional)

Once the happy-path works, run:

```bash
./scripts/install-launchagent.sh
```

This builds a release binary, installs it to `/usr/local/bin/smoothie-server`, and registers a LaunchAgent at `~/Library/LaunchAgents/dev.smoothie.server.plist` so the daemon starts at login.
